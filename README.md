# in.depth

Платформа подготовки к техническим интервью с лёгким RAG-поиском по JSON.

## Как устроены данные

- `content/questions/*.md` — исходные вопросы и подробные ответы. Каждый вопрос хранится в отдельном файле.
- `public/data/questions.json` — автоматически созданные данные для интерфейса. Не редактируйте файл вручную.
- `public/data/questions-index.json` — автоматически рассчитанные векторы для поиска.
- Пустой `languages` означает, что вопрос универсален и подходит для любого языка.
- Один вопрос может относиться сразу к нескольким языкам, ролям и компаниям.

После изменения Markdown выполните `npm run index`. Команда проверит обязательные поля, соберёт единый JSON и обновит RAG-индекс.

## Запуск

```bash
npm install
npm run index
npm run dev
```

`npm run dev` запускает Vite и маленький RAG API. Если API недоступен, интерфейс автоматически выполняет векторный поиск прямо в браузере — это позволяет публиковать проект как полностью статический сайт.

## Дешёвый статический хостинг

```bash
npm run index
npm run build
```

Папку `dist` можно разместить на любом статическом хостинге. База данных и постоянно работающий Node-сервер для этого режима не нужны.

## S3 и кэширование

Сайт загружает `questions.json` и `questions-index.json` из URL, заданного в `.env`. В браузере данные кэшируются на 5 минут; если S3 временно недоступен, используется последняя сохранённая копия или JSON из сборки.

После обновления Markdown соберите данные и загрузите два файла из `public/data` в каталог `data` бакета. Для них рекомендуется заголовок:

```text
Cache-Control: public, max-age=300, stale-while-revalidate=86400
```

Бакет должен разрешать публичный `GET` и CORS-запросы с домена сайта. Готовая CORS-конфигурация для публичных файлов находится в `scripts/s3-cors.xml`. Ключи S3 должны храниться только в настройках деплоя или локальном менеджере секретов — не в репозитории.

Обновление данных одной командой:

```bash
S3_ACCESS_KEY=... S3_SECRET_KEY=... npm run s3:upload
```
