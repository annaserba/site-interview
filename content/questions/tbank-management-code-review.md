## Короткий ответ

Минимум 2 ревьюера, автоматический CI + linting до ревью, чеклист для ревьюеров (безопасность, производительность, читаемость). Обратная связь — конструктивная, с примерами. Не блокер: если 2 одобрения — мержим.

## Контекст

Проверяют понимание процессов质量和 командной культуры.

## Как строить ответ

### Процесс

CI → lint → 2 ревьюера → одобрение → мерж. PR не больше 400 строк.

### Культура

Обратная связь с примерами: «предлагаю заменить X на Y, потому что Z».

### Автоматизация

Linter, formatter, unit-тесты — всё до ревью.

## Пример ответа

У нас PR проходит: 1) CI (lint, test, build), 2) минимум 2 одобрения, 3) changelog в PR. Ревьюеры используют чеклист: безопасность, производительность, читаемость, тесты. Мы ограничиваем PR 400 строками — больше很难.review. Обратная связь: «в строке 42 предлагаю X, потому что Y». Результат: снижение багов в production на 60%.

## Частые ошибки

- 1 ревьюер и формальное одобрение
- PR на 2000 строк
- «Всё ок» без конкретики
- Не автоматизировать lint/test
