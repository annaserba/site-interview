## Короткий ответ

Standards → code review → testing → metrics. Важно: баланс между speed и quality, automation.

## Контекст

Проверяют понимание quality assurance и его importance.

## Как строить ответ

### Standards

Какие standards: coding guidelines, best practices.

### Процессы

Какие процессы: code review, testing, CI/CD.

### Metrics

Что измеряете: code coverage, bug rate, satisfaction.

## Пример ответа

Standards: coding guidelines, linting, best practices. Процессы: code review, unit tests, integration tests, CI/CD. Metrics: code coverage (80%), bug rate (2 per sprint), satisfaction (4.5/5). Результат: ростquality, снижениеbugs на 50%.

## Частые ошибки

- Не having standards
- Not measuring quality
- Sacrificing quality for speed
- Not automating
