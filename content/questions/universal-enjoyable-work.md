## Короткий ответ

Назовите тип работы, где ваша мотивация совпадает с сильным результатом: разбирать неопределённую проблему, строить надёжную систему, превращать данные в решение или развивать инженеров. Подкрепите одним примером и свяжите с вакансией. Не делите работу на «любимую» и ту, которую выполнять не будете: покажите зрелое отношение к обязательной эксплуатации и рутине.

## Контекст

Вопрос универсален и проверяет устойчивую мотивацию и совпадение с реальным содержанием роли.

## Как строить ответ

### Назвать паттерн задач

Опишите проблему и роль, а не конкретную технологию.

### Дать доказательство

Приведите кейс, где интерес дал измеримый результат.

### Связать с ролью

Покажите, что понимаете баланс исследования, delivery и поддержки.


## Пример ответа

Мне нравится: 1) Solving complex technical problems — когда нужно найти нестандартное решение; 2) Seeing user impact — когда моя работа влияет на миллионы пользователей; 3) Mentoring — помогать junior'ам расти; 4) Learning new technologies — React Server Components, WebAssembly. Не нравится: бессмысленные meetings без agenda, micromanagement, bug fixes без возможности улучшить архитектуру. Пример: проект по оптимизации web vitals — я исследовал, нашёл bottleneck (unused JS 300KB), оптимизировал, измерил результат (LCP 4.5s → 1.2s). Этот цикл от исследования до результата — то, что меня мотивирует.

## Частые ошибки

- Отвечать общим «люблю сложные задачи».
- Называть только greenfield-разработку.
- Обесценивать рутинную часть работы.
