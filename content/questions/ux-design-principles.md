## Короткий ответ

Принципы: visibility of system status (показывайте что происходит), match between system and real world (понятный язык), user control and freedom (undo), consistency, error prevention, recognition over recall, flexibility and efficiency, aesthetic and minimalist design.

## Контекст

Фундаментальные heuristics Нильсена. Проверяют понимание UX principles.

## Как строить ответ

### 10 heuristics

1. Visibility of system status. 2. Match real world. 3. User control. 4. Consistency. 5. Error prevention. 6. Recognition > recall. 7. Flexibility. 8. Minimalist design. 9. Help recognize errors. 10. Help and documentation.

### Примеры

Loading indicators → visibility. Confirm dialogs → error prevention. Undo → user control.

## Пример ответа

Visibility: progress bar при загрузке. Real world: иконки корзины, папки. User control: кнопка "Назад". Consistency: одинаковые кнопки. Error prevention: disable unavailable buttons. Recognition: recent searches. Minimalist: только essential UI.

## Частые ошибки

- Игнорировать loading states
- Не давать feedback
- Делать complex navigation
- Не тестировать с реальными пользователями
