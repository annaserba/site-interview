## Короткий ответ

Используйте два указателя: левый движется от начала, правый — от конца. Каждый указатель пропускает небуквенные символы, затем буквы сравниваются в едином регистре. Каждый символ просматривается не более одного раза: время O(n), дополнительная память O(1). До кода зафиксируйте модель символов: для ASCII или кириллицы в BMP достаточно индексации строки, а полноценный Unicode требует отдельного решения для code points и case folding.

## Контекст

Нужно реализовать функцию для строк со смешанным регистром, пробелами и знаками препинания. Интервьюер ожидает решение без очищенной копии строки и без разворота, а также осознанное обсуждение границ Unicode.

## Как строить ответ

### Сначала определить контракт

Уточните, что считается буквой, какие алфавиты поддерживаются, должна ли пустая после фильтрации строка считаться палиндромом и требуется ли полная Unicode-корректность. В JavaScript строка индексируется UTF-16 code units, поэтому обещание «любой Unicode и O(1) памяти» нельзя давать без уточнений.

### Применить два указателя

Пока left меньше right, независимо сдвигайте каждый указатель до ближайшей буквы. Если нормализованные буквы различаются — сразу верните false; иначе сдвиньте оба указателя. После пересечения верните true. Это даёт ранний выход и не создаёт строку размера n.

### Отделить распознавание от обхода

Вынесите проверку буквы и нормализацию регистра в маленькие функции. Для оговорённого набора символов допустимы явные диапазоны; для Unicode — property escape `\p{L}` с флагом `u`, но отдельно проговорите суррогатные пары и символы, case folding которых меняет длину.

### Проверить инварианты и края

На каждой итерации всё за пределами диапазона left…right уже проверено. Пройдите мысленно пустую строку, одну букву, только пунктуацию, разные буквы по краям, смешанный регистр и несколько пропускаемых символов подряд.


## Код из интервью

```typescript
function isPalindrome(s: string): boolean {
  let left = 0;
  let right = s.length - 1;

  while (left < right) {
    while (left < right && !/[a-zA-Z0-9]/.test(s[left])) left++;
    while (left < right && !/[a-zA-Z0-9]/.test(s[right])) right--;

    if (s[left].toLowerCase() !== s[right].toLowerCase()) return false;

    left++;
    right--;
  }

  return true;
}

isPalindrome("A man, a plan, a canal: Panama"); // true
isPalindrome("race a car"); // false
```

## Пример ответа

Проверка палиндрома с two pointers:

```javascript
function isPalindrome(str) {
  let left = 0;
  let right = str.length - 1;

  while (left < right) {
    while (left < right && !/[a-zA-Z0-9]/.test(str[left])) left++;
    while (left < right && !/[a-zA-Z0-9]/.test(str[right])) right--;

    if (str[left].toLowerCase() !== str[right].toLowerCase()) {
      return false;
    }

    left++;
    right--;
  }

  return true;
}
```

Алгоритм: два указателя — с начала и конца. Пропускаем небуквенные символы через regex. Сравниваем без учёта регистра. Сложность: O(n) время, O(1) память. На практике: two pointers —ный подход для палиндромов, сравнения строк, поиска пар.

## Частые ошибки

- Сначала очищать и разворачивать строку, нарушая требование O(1) по памяти.
- Сдвигать указатели до проверки или забывать повторно пропустить несколько небуквенных символов подряд.
- Называть индексацию JavaScript-строки полноценной поддержкой Unicode.
- Использовать регулярное выражение без фиксации поддерживаемых алфавитов.
