## Короткий ответ

Не выбирайте инструмент абстрактно: покажите, какие задачи и масштабы решали каждым. SQL — извлечение, join, window functions, агрегации и работа близко к хранилищу; Python — статистика, автоматизация, API, сложные преобразования и воспроизводимые пайплайны. Назовите глубину, ограничения и пример, где осознанно перенесли вычисление между БД и Python ради стоимости, скорости или поддерживаемости.

## Контекст

Проверяется реальная глубина и умение выбирать место вычисления.

## Как строить ответ

### Дать факты

Назовите годы, типы задач, объёмы данных и production-ответственность.

### Показать границу

Тяжёлую фильтрацию и агрегацию обычно выгодно делать в БД, а статистическую модель — в Python.

### Признать пробелы

Честно обозначьте менее сильную область и конкретный план развития.

## Частые ошибки

- Отвечать одним названием инструмента.
- Перетаскивать сырые данные в pandas без необходимости.
- Называть notebook production-пайплайном.
