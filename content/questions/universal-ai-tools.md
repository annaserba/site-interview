## Короткий ответ

Назовите конкретные инструменты и свяжите каждый с рабочей задачей: «ChatGPT для исследования API и генерации boilerplate, Copilot в IDE для автодополнения, Claude для ревью архитектурных решений». Покажите осознанный подход — что используете, зачем именно этот инструмент и какие у него ограничения.

## Контекст

Проверяется digital literacy, умение внедрять AI в рабочий процесс и критическое отношение к результатам генерации.

## Как строить ответ

### Перечислить инструменты по задачам

Каждый инструмент привяжите к конкретному use-case: генерация кода, отладка, работа с данными, документация, архитектура.

### Объяснить критерии выбора

Почему этот инструмент для этой задачи: скорость, качество кода, удобство интеграции, стоимость.

### Показать критическое отношение

Где нейросети ошибаются в вашей области и как вы это проверяете (тесты, ручное ревью, сравнение версий).

## Пример ответа

Я активно использую AI-инструменты: 1) GitHub Copilot — для генерации boilerplate кода, unit-тестов, экономит ~30% времени; 2) ChatGPT — для исследования API, отладки, составления RFC; 3) Claude — для code review, рефакторинга, объяснения legacy кода. Важно: AI — это инструмент, а не замена. Я всегда проверяю результат: AI может генерировать неверный код. На практике: 60% времени — мой код, 30% — AI-ассистированный, 10% — ручная отладка. Не использую AI для security-critical кода (auth, payments).

## Частые ошибки

- Говорить «всеми» без конкретики.
- Утверждать, что нейросети не нужны и вы пишете всё сами.
- Не упоминать проверку и валидацию сгенерированного кода.
- Хвалить инструмент без указания его слабых сторон.
