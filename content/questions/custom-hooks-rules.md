## Короткий ответ

Правила: 1) Название с префиксом use. 2) Вызываться на верхнем уровне (не в условиях, циклах, вложенных функциях). 3) Только из React-компонентов или других хуков. Кастомные хуки — переиспользование логики с хуками (state, эффекты, контекст).

## Контекст

Интервьюер проверяет знание правил хуков и умение создавать переиспользуемую логику.

## Как строить ответ

### Префикс use

Обязательно для проверки правил React (eslint-plugin-react-hooks).

### Уровень вызова

Хуки вызываются на верхнем уровне. Это обеспечивает порядок вызовов между рендерами.

### Переиспользование логики

Кастомный хук — функция с хуками внутри. Возвращает данные/функции, которые нужны компоненту.

### Тестируемость

Кастомные хуки легко тестировать отдельно от компонентов.

## Пример ответа

Правила хуков: 1) Начинать с use (useAuth, useFetch). 2) Вызывать на верхнем уровне — не в if/for/вложенных функциях. 3) Только из функциональных компонентов или других хуков. Кастомные хуки позволяют вынести общую логику: useFetch для запросов, useLocalStorage для хранения, useDebounce для отсрочки. Внутри могут использовать useState, useEffect и другие хуки.

## Частые ошибки

- Нарушать правило верхнего уровня (вызывать в условии)
- Не использовать префикс use
- Создавать хуки для одного компонента (не переиспользование)
- Забывать возвращать нужные данные из хука
