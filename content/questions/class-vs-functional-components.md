## Короткий ответ

Классовые: наследуют от React.Component, имеют state и lifecycle методы (componentDidMount, render). Функциональные: простые функции, используют хуки (useState, useEffect). Современный подход — функциональные + хуки. Классовые используются в legacy-кодах.

## Контекст

Интервьюер проверяет понимание двух парадигм React и почему хуки стали стандартом.

## Как строить ответ

### Классовые компоненты

Наследование от React.Component. this.state, this.setState. Жизненный цикл через методы.

### Функциональные компоненты

Простые функции. Хуки: useState, useEffect, useContext. Нет this.

### Хуки

Позволяют использовать state и lifecycle в функциональных компонентах. Переиспользование логики.

### Современный подход

Функциональные + хуки. Классовые — только для legacy или специфических случаев.

## Пример ответа

Классовые компоненты — это ES6 классы с методом render(). Имеют state, жизненный цикл (componentDidMount, shouldComponentUpdate, componentWillUnmount). Функциональные — это функции, которые возвращают JSX. Используют хуки для state (useState) и побочных эффектов (useEffect). Преимущества функциональных: проще тестировать, нет this, легче переиспользовать логику через хуки. Классовые всё ещё встречаются в legacy-проектах.

## Частые ошибки

- Путать this в классовых компонентах
- Использовать хуки условно (не в корне компонента)
- Не понимать разницу жизненного цикла
- Думать, что классовые быстрее функциональных
