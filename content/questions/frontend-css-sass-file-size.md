## Короткий ответ

Назовите типичный размер одного модуля (200–600 строк), размер всей кодовой базы стилей (5–40 тысяч строк), методы борьбы с раздуванием: разбиение по компонентам, scoped-стили, удаление мёртвого кода, критический CSS и инструменты измерения бандла. Подчеркните, что важнее не строки, а архитектура и переиспользование.

## Контекст

Проверяют масштаб проектов, с которыми кандидат работал, и понимание проблем больших кодовых баз стилей.

## Как строить ответ

### Назвать конкретные цифры

Опишите крупнейший проект: общий объём, структура (файлы/модули), как организованы.

### Объяснить подход к архитектуре

ITCSS, БЭМ, CSS Modules, utility-first — какой подход использовали и почему.

### Рассказать про оптимизацию

Как боролись с ростом: dead code elimination, критический CSS, code splitting стилей, миграция на CSS-in-JS или Tailwind.

## Частые ошибки

- Называть только общее число без разбивки по модулям.
- Не упоминать инструменты анализа (webpack-bundle-analyzer, coverage в DevTools).
- Говорить «файлы были маленькие» без объяснения архитектурных решений.

## Дополнительные вопросы

- Как вы организовывали переиспользуемые стили?
- Приходилось ли мигрировать с одной методологии на другую?
- Как измеряли влияние стилей на производительность?
- Работали ли с critical CSS и как внедряли?
