## Короткий ответ

Learning sessions → documentation → pair programming → code review. Важно: knowledge sharing, неbus factor.

## Контекст

Интервьюер хочет понять, как вы đảm знание стека в команде.

## Как строить ответ

### Learning

Какlearning-аете: sessions, workshops, courses.

### Documentation

Как документируете: wiki, ADR, README.

### Sharing

Как sharing-аете: pair programming, code review.

## Пример ответа

Learning: weekly learning sessions, workshops. Documentation: wiki, ADR, README. Sharing: pair programming, code review. Результат: knowledge distributed, bus factor reduced.

## Частые ошибки

- Not sharing knowledge
- Not documenting
- Not learning
- Bus factor
