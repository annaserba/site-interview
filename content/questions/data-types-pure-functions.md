## Короткий ответ

Примитивы: string, number, bigint, boolean, undefined, null, symbol. Ссылочные: object (включая array, function, Date, RegExp, Map, Set). Чистые функции: всегда возвращают один и тот же результат при одинаковых аргументах, не имеют побочных эффектов.

## Контекст

Интервьюер проверяет базовые знания JavaScript и понимание концепции чистых функций.

## Как строить ответ

### Примитивы vs Ссылочные

Примитивы копируются по значению. Объекты — по ссылке. Поэтому мутировать объект опасно.

### Чистые функции

Детерминированы + нет побочных эффектов. Легко тестировать, кешировать, использовать в React.

### Побочные эффекты

Мутации внешнего state, fetch, console.log, setTimeout — всё это побочные эффекты.

### Иммутабельность

Чистые функции работают с иммутабельными данными. Не мутируют входные аргументы.

## Пример ответа

Типы данных: Примитивы — string, number, boolean, null, undefined, symbol, bigint. Ссылочные — object (массивы, функции, даты — все объекты). Чистые функции: 1) детерминированы (одинаковый вход = одинаковый выход), 2) не имеют побочных эффектов (не мутируют внешний state, не делают fetch). Пример чистой: const add = (a, b) => a + b. Нечистая: let count = 0; const inc = () => count++.

## Частые ошибки

- Путать примитивы и ссылочные типы
- Мутировать аргументы в чистых функциях
- Считать функции с console.log чистыми
- Не учитывать, что object включает array и function
