## Короткий ответ

Лексическое окружение — область видимости, определяемая местом объявления переменной. Замыкание — функция вместе со ссылкой на внешнее лексическое окружение, в котором она была создана. Замыкания позволяют функциям «помнить» переменные из внешних областей видимости.

## Контекст

Интервьюер проверяет понимание одного из ключевых концепций JavaScript.

## Как строить ответ

### Лексическая область видимости

Область видимости определяется местом написания кода, а не местом вызова.

### Замыкание

Функция + ссылка на внешнее лексическое окружение. Работает даже после завершения внешней функции.

### Цепочка окружений

Каждое окружение ссылается на родительское. Поиск переменных идёт по цепочке.

### Практическое применение

Инкапсуляция, data privacy, функциональное программирование, partial application.

## Пример ответа

Лексическое окружение создаётся при вызове функции и содержит её локальные переменные + ссылку на внешнее окружение. Замыкание возникает, когда внутренняя функция ссылается на переменные внешней функции после того, как внешняя функция завершилась. Пример: function outer() { let x = 10; return function inner() { return x; }; } const close = outer(); close() // 10 — inner «помнит» x.

## Частые ошибки

- Путать лексическую и динамическую область видимости
- Думать, что замыкание — это только функция внутри функции
- Не понимать, что замыкание хранит ссылку, а не копию
- Забывать про утечки памяти при неправильном использовании
