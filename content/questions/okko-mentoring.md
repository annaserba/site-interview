## Короткий ответ

Опишите исходный уровень и цель человека, а не количество созвонов. Составьте план с рабочими задачами возрастающей сложности, регулярной обратной связью и критериями самостоятельности. Не решайте задачу вместо менти: помогайте формулировать гипотезы, давайте контекст и своевременно повышайте сложность. Результат измеряется временем до самостоятельной доставки и качеством решений.

## Контекст

Проверяется способность масштабировать знания, а не только отвечать на вопросы.

## Как строить ответ

### Диагностика

Навыки, пробелы, цель и доступное время.

### Практика

Задачи, code review, парная работа и разбор решений.

### Результат

Самостоятельность, качество и следующая зона роста.

## Пример ответа

Я менторил двух младших разработчиков. Подход: 1) Weekly 1:1 — 30 минут, обсуждаем blockers и goals; 2) Pair programming — совместно пишем сложные фичи; 3) Code review — подробные комментарии, объясняю «почему»; 4) Learning plan — составили roadmap на 3 месяца. Результат: один стажёр через 3 месяца стал джуниором, второй — через 4. Самый важный инсайт: не давать готовых ответов, а задавать вопросы: «Как бы ты решил?». Также важно: дать возможность ошибаться в безопасной среде — на test/staging.

## Частые ошибки

- Называть менторством только onboarding.
- Давать готовые ответы на каждую проблему.
- Не собирать обратную связь у менти.
