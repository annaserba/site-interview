## Короткий ответ

TypeScript — это JavaScript с типами. Преимущества: раннее обнаружение ошибок, лучшая поддержка IDE (автодополнение, рефакторинг), самодокументируемый код, безопасность при рефакторинге, поддержка generics, utility types, decorators. Особенности: компиляция в JS, строгая типизация, интерфейсы, типы.

## Как строить ответ

### Статическая типизация

Типы проверяются на этапе компиляции. Ловит ошибки до запуска кода.

### Интерфейсы и типы

Описывают структуру объектов. Интерфейсы расширяемы, типы — гибче.

### Generics

Параметризованные типы. Позволяют писать переиспользуемый типизированный код.

### Типы стираются

TypeScript — это надмножество JS. В рантайме типов нет, всё компилируется в JS.

## Пример ответа

TypeScript добавляет статическую типизацию к JavaScript. Преимущества: 1) Ошибки ловятся на этапе компиляции, а не в рантайме. 2) IDE знает типы — автодополнение, навигация, рефакторинг. 3) Код становится документацией. 4) Generics, utility types, conditional types для гибкой типизации. 5) Интерфейсы для описания структуры объектов. Особенности: типы стираются при компиляции, any — escape hatch, type assertions.

## Частые ошибки

- Злоупотреблять any — теряется benefit типизации
- Не использовать strict mode
- Путать интерфейсы и типы
- Думать, что TypeScript защищает от всех ошибок
