## Короткий ответ

Индивидуальный план развития: цели на квартал, регулярные 1:1, code review как learning opportunity. Не делать за них — направлять. Менторство через вопросы: «почему ты так сделал?».pair programming для сложных задач.

## Контекст

Проверяют способность расти через других, а не только через свой код.

## Как строить ответ

### Подход

Не делать за них, а направлять через вопросы.

### Инструменты

Code review, pair programming, learning goals.

### Метрики

Рост компетенций: от «пишет код» до «решает задачи».

## Пример ответа

Для junior завёл learning path: первые 3 месяца — наставник, pair programming, code review с объяснениями. Месяц 4-6 — ownership, ментор как reviewer. Месяц 7+ — feature delivery. Цели на квартал: «научиться писать тесты», «понять domain». Результат: 80% junior вырастают до middle за 12 месяцев.

## Частые ошибки

- Делать за них
- Не давать feedback
- Не ставить цели
- Один формат для всех

## Дополнительные вопросы

- Как мотивируете на скучные задачи?
- Как обрабатываете сопротивление?
- Как измеряете прогресс?
