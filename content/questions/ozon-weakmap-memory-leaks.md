## Короткий ответ

WeakMap/WeakSet хранят слабые ссылки на ключи — сборщик мусора может удалить объект-ключ, даже если он есть в WeakMap. Это предотвращает утечки памяти при кэшировании и хранении метаданных: когда объект больше нигде не используется, его запись автоматически удаляется.

## Контекст

Интервьюер проверяет понимание разницы между Map/Set и WeakMap/WeakSet, знание как работает garbage collection со слабыми ссылками и умение применять это в практических сценариях.

## Как строить ответ

### Объяснить разницу между Map/Set и WeakMap/WeakSet

Map/Setдерживают сильные ссылки на ключи — пока запись есть в коллекции, объект не может быть собран. WeakMap/WeakSet допускают только объекты как ключи и не препятствуют GC: если на объект нет других ссылок, он удаляется.

### Описать как работает сборщик мусора

JavaScript использует reachability-based GC: объект жив, пока достижим из корня (стек, глобалы, активные таймеры). WeakMap не увеличивает reachability ключа — GC может удалить ключ, и запись исчезнет автоматически.

### Показать практические сценарии

Кэширование результатов вычислений, хранение метаданных DOM-элементов, привязка данных к объектам без увеличения ссылочного счётчика.

### Разобрать ограничения

WeakMap не итерируем (нет .keys(), .values(), .entries()), нет .size — это сделано намеренно для предсказуемого поведения GC.

## Код из интервью

```javascript
// Кэширование результатов с WeakMap
const cache = new WeakMap();

function processHeavy(obj) {
  if (cache.has(obj)) {
    return cache.get(obj);
  }
  const result = obj.data.reduce((sum, n) => sum + n * 2, 0);
  cache.set(obj, result);
  return result;
}

const input = { data: [1, 2, 3] };
processHeavy(input); // вычисляем и кэшируем

// Когда input больше не нужен, GC удалит и запись в cache
let cacheEntry = cache.get(input); // undefined
```

## Пример ответа

WeakMap и WeakSet хранят слабые ссылки на ключи. В отличие от Map, пока объект-ключ существует где-то в программе, запись доступна. Но если на объект нет других ссылок — GC может удалить его, и запись из WeakMap тоже исчезнет.

Это полезно для кэширования: если объект удалён из DOM, его кэшированные результаты тоже освобождают память без ручной очистки. Я использую это в SPA для хранения метаданных компонентов — не нужно отписывать cleanup в useEffect.

## Частые ошибки

- Считать, что WeakMap автоматически очищается по таймеру — это делает GC.
- Пытаться итерировать WeakMap или получить .size — их нет.
- Использовать примитивы как ключи WeakMap — допустимы только объекты.
- Путать WeakMap с "приватными полями" — WeakMap хранит внешние данные, а не приватные поля класса.
