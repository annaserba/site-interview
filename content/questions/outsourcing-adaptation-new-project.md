## Короткий ответ

Формат STAR: Situation — новый проект с незнакомым стеком, Task — быстрое включение в команду, Action — документация (официальная + внутренняя), pair programming с тимлидом, изучение codebase через фичи, Result — полная вовлечённость за 2-3 недели. Ключ — умение быстро учиться и не бояться задавать вопросы.

## Контекст

Интервьюер ожидает понимания процесса обучения, адаптации к новому стеку, работы в команде и self-management.

## Как строить ответ

### STAR формат

Опишите конкретный пример: какой стек был незнаком, как долго адаптировались, что помогло.

### Learning strategy

Официальная документация → tutorials → codebase existing features → pair programming. Не пытайтесь изучить всё сразу.

### First contributions

Начинайте с малого: bug fix, documentation, small feature. Это даёт понимание workflow (git flow, CI/CD, code review).

### Pair programming

Спросите у тимлида 1-2 pair programming sessions. Это быстрее, чем часами читать код вслепую.

### Asking questions

Задавайте вопросы, но сначала попробуйте найти ответ самостоятельно. Фиксируйте ответы в заметки — это поможет следующему новичку.

## Пример ответа

Situation: Пришёл на проект на React + GraphQL + Apollo, хотя до этого работал с Vue + REST. Стек был полностью незнаком, кодовая база — 200K строк.

Task: Необходимо было начать закрывать задачи через 2-3 недели, не замедляя команду.

Action: Первые 3 дня изучал official docs (React, Apollo), затем — internal wiki с архитектурными решениями. Попросил у тимлида 2 pair programming session: одна по фиче (понял workflow), вторая по деплою (понял CI/CD). Начал с малого: исправил typo в документации, затем fixed bug в формуле расчёта, затем взял небольшую фичу. Все вопросы фиксировал в Notion — получилась внутренняя "cheat sheet".

Result: Через 2 недели закрывал задачи самостоятельно, через месяц — менторил нового junior. Стек изучал постепенно: сначала понял общую архитектуру, потом детали.

## Частые ошибки

- Пытаться изучить весь стек за первый день — information overload.
- Не задавать вопросы, тратя часы на проблему, которую тимлид решит за 5 минут.
- Сразу рефакторить незнакомый код, не понимая контекст.
- Не фиксировать изученное — через неделю забываете, что нашли.
- Сравнивать новый стек со старым («во Vue это проще») вместо изучения новых подходов.
