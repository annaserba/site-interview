## Короткий ответ

Server Components рендерятся на сервере и отправляют клиенту только результат (не код). Это уменьшает bundle size и ускоряет первую отрисовку. В e-commerce критично для SEO: контент товаров индексируется без client-side рендеринга, а клиентские интеракции работают через Client Components.

## Контекст

Интервьюер проверяет понимание архитектуры RSC, разницу между серверными и клиентскими компонентами, ограничения сериализации и практическое применение в e-commerce платформах.

## Как строить ответ

### Объяснить серверные vs клиентские компоненты

Server Components выполняются только на сервере, не отправляют JS на клиент. Client Components (с "use client") работают как обычные React-компоненты, поддерживают state и effects.

### Описать как передаются данные

Серверный компонент сериализует результат (JSX tree) в специальный формат, клиент десериализует и рендерит. Пропсы должны быть сериализуемыми — нет функций, нет классов.

### Показать ограничения

Нельзя использовать useState/useEffect в серверных компонентах, нет доступа к browser APIs, нет обработчиков событий.

### Разобрать гидратацию

Клиентские компоненты "гидрируются" — React восстанавливает state и привязывает обработчики. Серверные не гидрируются — они уже отрендерены.

### Практика в e-commerce

SEO-критичные страницы (каталог, карточка товара) — серверные компоненты. Интерактивные элементы (корзина, фильтры, избранные) — клиентские.

## Код из интервью

```tsx
// Серверный компонент — данные берутся напрямую из БД
async function ProductPage({ params }: { params: { id: string } }) {
  const product = await db.product.findUnique({ where: { id: params.id } });

  return (
    <div>
      <h1>{product.name}</h1>
      <p>{product.price}</p>
      <AddToCartButton productId={product.id} />
    </div>
  );
}

// Клиентский компонент — интерактивность
"use client";
function AddToCartButton({ productId }: { productId: string }) {
  const [loading, setLoading] = useState(false);

  return (
    <button
      disabled={loading}
      onClick={async () => {
        setLoading(true);
        await fetch("/api/cart", {
          method: "POST",
          body: JSON.stringify({ productId }),
        });
        setLoading(false);
      }}
    >
      {loading ? "Добавляем..." : "В корзину"}
    </button>
  );
}
```

## Пример ответа

React Server Components — это компоненты, которые рендерятся только на сервере и не добавляют JS в bundle. Серверный компонент получает данные, рендерит JSX и отправляет клиенту "снимок" — клиент только десериализует, не гидрирует.

В e-commerce это критично: страница товара с серверным компонентом отдаёт полностью отрендеренный HTML для SEO, а интерактивность (кнопки, фильтры, корзина) — через клиентские компоненты. Bundle уменьшается на весь код серверных компонентов, время первой отрисовки снижается.

Ограничение: пропсы и данные должны быть сериализуемы — нельзя передать функцию или экземпляр Date из серверного в клиентский компонент.

## Частые ошибки

- Путать SSR и RSC — SSR рендерит компонент на сервере, но клиент получает JS и гидрирует. RSC — нет JS на клиенте.
- Считать, что RSC заменяет SSR — они дополняют друг друга.
- Пытаться использовать hooks в серверных компонентах — это ошибка компиляции.
- Не учитывать, что данные серверного компонента кэшируются — при частых обновлениях данных нужен revalidate.
