## Короткий ответ

Изучаю через практику: ставлю мини-проект, читаю документацию, смотрю доклады. Использую AI-инструменты (ChatGPT, Claude) для ускорения. Технологию считаю освоенной, когда могу объяснить её коллеге или написать работающий прототип.

## Контекст

HR проверяет способность к самообучению и адаптации. Важно показать систему, а не просто "читаю статьи".

## Как строить ответ

### Метод
1. **Практика прежде теории** — ставлю конкретную задачу: «написать Todo на новом фреймворке» или «мигрировать модуль»
2. **Документация + доклады** — официальные доки + конференции (YouTube: HolyJS, FrontendConf, HighLoad)
3. **AI-ассистенты** — ChatGPT для объяснения концепций, Claude Code для рефакторинга, DeepSeek для sensitive-проектов
4. **Пет-проекты** — изолированная среда без риска сломать прод
5. **Объяснить другому** — пишу заметки или рассказываю коллеге (метод Фейнмана)

### Критерий освоения
Технология освоена, когда могу:
- Написать работающий прототип без подглядывания в туториал
- Объяснить ключевые концепции junior-разработчику
- Понимать trade-offs и когда НЕ использовать эту технологию

## Пример ответа

Обычно начинаю с практической задачи. Например, когда изучал Go — взял небольшой микросервис, который хорошо знал на Python, и переписал его. За первые два дня разобрался с синтаксисом, горутинами и стандартной библиотекой. Документация Go отличная — читал её параллельно.

Дальше смотрю доклады на YouTube: HighLoad, GolangConf — там разбирают реальные кейсы, а не toy-примеры. AI использую для ускорения: ChatGPT объясняет незнакомые паттерны, Claude Code помогает с рефакторингом.

Технологию считаю освоенной после пет-проекта в прод-подобных условиях: с логами, тестами, CI/CD. И когда могу провести код-ревью для коллеги на этой технологии.

## Частые ошибки

- Бесконечно читать теорию без практики
- Прыгать между технологиями, не доводя ни одну до рабочего состояния
- Копипастить AI-код без понимания
- Не проверять знания на реальной задаче

## Дополнительные вопросы

- Расскажите про последнюю технологию, которую вы освоили. Какой был процесс?
- Как вы решаете, какую технологию стоит изучить, а какую — нет?
- Что делаете, когда застряли в изучении?
- Как оцениваете качество обучающих материалов?
