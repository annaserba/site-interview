## Короткий ответ

Структурированный onboarding: первая неделя — инфраструктура и инструменты, вторая — domain knowledge, третья — first task. Buddy-система. Не бросать в cold water. Regular check-in с manager.

## Контекст

Проверяют понимание: онбординг — это инвестиция в retention и скорость.

## Как строить ответ

### Неделя 1

Инфраструктура, инструменты, доступы, process overview.

### Неделя 2

Domain knowledge, архитектура, кодовая база.

### Неделя 3+

First task с mentorship, code review, feedback.

## Пример ответа

3-недельный onboarding: неделя 1 — setup environment,meet team, understand process; неделя 2 — domain deep-dive, архитектура, code base tour; неделя 3 — first feature с buddy. Buddy — senior developer, помогает с вопросами. Regular check-in с manager: end of week 1, 2, 3. Результат: time-to-first-commit сократился с 3 недель до 5 дней, retention нового найма — 95%.

## Частые ошибки

- Нет структуры
- Нет buddy
- Слишком много информации сразу
- Не check-in
