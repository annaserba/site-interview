## Короткий ответ

async/await — синтаксический сахар над промисами. async-функция всегда возвращает промис. await приостанавливает выполнение до завершения промиса. Преимущество: читаемый код, обработка ошибок через try/catch, отладка. Отличие от .then: sequential vs parallel.

## Контекст

Интервьюер проверяет знание современного синтаксиса асинхронного кода.

## Как строить ответ

### Сахар синтаксиса

async/await не заменяет промисы, а упрощает работу с ними. Под капотом — тот же .then.

### Последовательность

Await блокирует выполнение до завершения промиса. Если промисы независимые — используйте Promise.all.

### Обработка ошибок

try/catch для ошибок. Можно комбинировать с .catch() для гибкости.

### Циклы и async

В циклах forEach await не работает. Используйте for...of или Promise.all для параллельности.

## Пример ответа

async/await — это синтаксис для работы с промисами. async-функция возвращает промис. Await «приостанавливает» выполнение функции до получения результата промиса. Преимущества перед .then: 1) Код выглядит синхронным. 2) Ошибки ловятся через try/catch. 3) Легче отлаживать. Но: если промисы не зависят друг от друга, лучше использовать Promise.all для параллельного выполнения.

## Частые ошибки

- Использовать await в forEach
- Не обрабатывать ошибки через try/catch
- Писать последовательные await когда можно параллельно
- Забывать, что async-функция возвращает промис
