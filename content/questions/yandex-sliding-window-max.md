## Короткий ответ

Задача — найти максимум в каждом окне размера k по массиву. Brute force O(n*k) проверяет каждое окно отдельно. Оптимальное решение O(n) использует monotonic deque — дек, элементы в котором убывают от головы к хвосту. При движении окна удаляем элементы, вышедшие за границу, и поддерживаем убывающий порядок.

## Контекст

Это классическая алгоритмическая задача на собеседованиях в Яндексе. Интервьюер ожидает не только решение, но и анализ сложности, понимание為什麼 deque работает, и умение объяснить monotonic property.

## Как строить ответ

### Brute force решение

Для каждого окна [i, i+k-1] находим максимум перебором. Сложность O(n*k) по времени, O(1) по памяти. При n=10^5 и k=10^4 будет 10^9 операций — слишком медленно.

### Monotonic deque — идея

Deque хранит индексы элементов в убывающем порядке значений. Голова deque — всегда максимум текущего окна. Ключевое свойство: если элемент A меньше элемента B и A правее B, то A никогда не будет максимумом, пока B в окне.

### Поддержка deque

При добавлении нового элемента: удаляем все элементы с хвоста, которые меньше нового (они бесполезны). Добавляем новый элемент в хвост. При удалении из окна: если голова deque — индекс, вышедший за границу, удаляем с головы.

### Анализ сложности

Каждый элемент добавляется в deque один раз и удаляется один раз. Итого O(n) по времени. Память O(k) — размер deque не превышает размер окна.

### Edge cases

Пустой массив, k=1 (каждый элемент — максимум), k=n (максимум всего массива), все элементы одинаковые.

## Код из интервью

```javascript
function maxSlidingWindow(nums, k) {
  const deque = []; // индексы, значения убывают
  const result = [];

  for (let i = 0; i < nums.length; i++) {
    // Удаляем элементы, вышедшие за границу окна
    while (deque.length && deque[0] <= i - k) {
      deque.shift();
    }

    // Удаляем с хвоста все элементы меньше текущего
    while (deque.length && nums[deque[deque.length - 1]] <= nums[i]) {
      deque.pop();
    }

    deque.push(i);

    // Начинаем записывать результат, когда окно полностью заполнено
    if (i >= k - 1) {
      result.push(nums[deque[0]]);
    }
  }

  return result;
}

// Python
function maxSlidingWindowPython(nums, k) {
  const deque = [];
  const result = [];

  for (let i = 0; i < nums.length; i++) {
    while (deque.length && deque[0] <= i - k) {
      deque.shift();
    }
    while (deque.length && nums[deque[deque.length - 1]] <= nums[i]) {
      deque.pop();
    }
    deque.push(i);
    if (i >= k - 1) {
      result.push(nums[deque[0]]);
    }
  }
  return result;
}
```

## Пример ответа

Начнём с brute force: для каждого окна находим максимум за O(k). Итого O(n*k) — не проходит для больших входов.

Оптимизация через monotonic deque: дек хранит индексы в убывающем порядке значений. Голова — всегда максимум. При каждом шаге: удаляем устаревшие индексы (вышли за границу k), удаляем с хвоста всё, что меньше текущего элемента (эти элементы не будут максимумом), добавляем текущий.

Почему O(n): каждый элемент добавляется в deque один раз и удаляется один раз. deque не может содержать больше k элементов. Запись результата — O(1), берём значение по голове deque.

## Частые ошибки

- Использовать Math.max(...window) для каждого окна — O(n*k), не проходит на больших данных.
- Не удалять устаревшие элементы с головы deque — максимум будет неправильным.
- Путать порядок удаления: с головы — устаревшие, с хвоста — меньшие текущего.
- Не проверять k=1 и k=n — edge cases, которые могут сломать логику.
