## Короткий ответ

Браузер загружает HTML, строит DOM-дерево, параллельно загружает CSS и JS. CSSOM + DOM = Render Tree. Layout → Paint → Composite. Ключевые этапы: парсинг HTML, построение DOM, загрузка ресурсов, вычисление стилей, построение Render Tree, layout, paint, composite.

## Контекст

Интервьюер проверяет понимание базового процесса рендеринга в браузере и как оптимизировать загрузку.

## Как строить ответ

### DOM и CSSOM

Браузер парсит HTML в DOM-дерево и CSS в CSSOM. Эти две структуры объединяются в Render Tree.

### Layout, Paint, Composite

Render Tree проходит три этапа: layout (вычисление размеров и позиций), paint (отрисовка), composite (объединение слоёв).

### Блокировка рендеринга

CSS блокирует рендеринг до загрузки всех стилей. JS блокирует парсинг HTML если нет async или defer.

### Критический путь

Оптимизация критического пути — минимизация времени до первого рендеринга (FCP, LCP).

## Пример ответа

При загрузке страницы браузер: 1) Парсит HTML и строит DOM-дерево. 2) Параллельно загружает CSS и JS файлы. 3) Строит CSSOM из CSS. 4) Объединяет DOM и CSSOM в Render Tree. 5) Вычисляет layout (позиции и размеры элементов). 6) Paint — отрисовывает пиксели. 7) Composite — объединяет слои. Важно: CSS блокирует рендеринг, JS может блокировать парсинг HTML.

## Частые ошибки

- Утверждать, что DOM и Render Tree — одно и то же, пропуская этап отсеивания display:none и псевдоэлементов.
- Считать, что JavaScript не влияет на скорость отрисовки первого кадра, игнорируя блокировку парсера HTML.
- Упускать разницу между атрибутами async и defer при подключении внешних скриптов.
- Не связывать понятие критического пути рендеринга с конкретными метриками Web Vitals: FCP, LCP, TBT.
