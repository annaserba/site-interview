## Короткий ответ

Структурированное интервью: coding (40 мин), system design (30 мин), behavioral (20 мин). Чеклист для каждого этапа. Оценка по 4 критериям: technical depth, problem solving, communication, culture fit. Декомпрессия с командой перед решением.

## Контекст

Проверяют зрелость процесса найма: понимает ли кандидат, что интервью — это не проверка знаний, а оценка potential.

## Как строить ответ

### Структура

Опишите конкретный формат: этапы, время, критерии оценки.

### Критерии

Четкие, измеримые: не «понравился», а «решает задачи за X минут, объясняет trade-offs».

### Команда

Вовлекайте team members — они видят то, что вы не замечаете.

## Пример ответа

У нас 3 этапа: coding (algos + domain), system design, behavioral. Coding — 40 минут, 2 задачи. System design — реальный архитектурный challenge из нашего домена. Behavioral — STAR-формат, 3 вопроса. После — декомпрессия: каждый интервьюер заполняет чеклист (1-5 по 4 критериям). Решение — голосование. Результат: снижение false positive на 35% за квартал.

## Частые ошибки

- Интервью без структуры
- Оценивать «симпатию» вместо компетенций
- Не делиться обратной связью с кандидатом
- Один человек решает судьбу кандидата
