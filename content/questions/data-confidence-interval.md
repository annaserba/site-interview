## Короткий ответ

Доверительный интервал — результат процедуры, которая при многократном повторении выборки накрывает истинный параметр с заданной частотой, например 95%. Это не вероятность 95%, что уже рассчитанный фиксированный интервал содержит параметр. Метод зависит от статистики, дизайна выборки и допущений; при кластерах, тяжёлых хвостах или последовательных просмотрах обычная формула может давать неверное покрытие.

## Контекст

Нужно объяснить интерпретацию, способ расчёта и связь ширины интервала с шумом и размером выборки.

## Как строить ответ

### Назвать оцениваемый параметр

Среднее, доля, разность метрик или uplift требуют разных оценок стандартной ошибки.

### Проверить дизайн

Учитывайте независимость, кластеры, стратификацию, repeated measures и распределение метрики; при необходимости используйте bootstrap или robust SE.

### Интерпретировать практически

Сопоставьте интервал с нулём и минимально значимым эффектом, а не только с p-value.


## Код из интервью

```python
from scipy import stats
import numpy as np

# A/B тест: t-test
control = np.random.normal(100, 15, 1000)
treatment = np.random.normal(105, 15, 1000)

stat, p_value = stats.ttest_ind(control, treatment)
print(f"t-statistic: {stat:.3f}, p-value: {p_value:.4f}")

# Доверительный интервал разницы
diff = treatment.mean() - control.mean()
se = np.sqrt(control.var()/len(control) + treatment.var()/len(treatment))
ci_low, ci_high = diff - 1.96*se, diff + 1.96*se
print(f"95% CI: [{ci_low:.2f}, {ci_high:.2f}]")
```

## Пример ответа

Доверительный интервал — это диапазон значений, в который с заданной вероятностью (обычно 95%) попадает истинное параметрическое значение. Например, средний чек = 1200 руб., 95% CI = [1150, 1250]. Это означает, что если мы многократно будем строить такие интервалы, в 95% случаев истинное среднее попадёт в этот диапазон. Важно: это не вероятность того, что истинное среднее в конкретном интервале. На практике я использую CI для сравнения групп: если CI для разницы не пересекает ноль, разница статистически значима. Также CI полезен для оценки размера эффекта.

## Частые ошибки

- Говорить о 95% вероятности параметра внутри готового frequentist-интервала.
- Игнорировать единицу рандомизации.
- Считать узкий интервал доказательством отсутствия систематической ошибки.
