## Короткий ответ

Reconciliation — процесс сравнения нового и старого Virtual DOM. Два типа узлов: разный тип → полная замена, одинаковый тип → обновление пропсов. Для списков key помогает React идентифицировать элементы и переиспользовать их, а не пересоздавать.

## Контекст

Интервьюер проверяет понимание внутреннего устройства React и как писать эффективный код.

## Как строить ответ

### Diffing

Сравнение двух деревьев за O(n). Эвристика: элементы одного типа — обновляются, разных — заменяются.

### Роль key

Уникальный идентификатор для элемента в списке. Помогает React определить переиспользование.

### Index как key

Использовать index как key опасно при изменении порядке или удалении элементов.

### Иммутабельность

Не мутируйте данные — React сравнивает по ссылке. Новые объекты/массивы = новый рендер.

## Пример ответа

Алгоритм reconciliation: 1) Если тип элемента изменился — старый удаляется, создаётся новый. 2) Если тип тот же — обновляются пропсы, рекурсивно сравниваются дети. 3) Для списков key позволяет React понять, какие элементы были добавлены, удалены или перемещены. Без key React сравнивает по индексу → ошибки при изменении порядка.

## Частые ошибки

- Использовать index как key при динамических списках
- Мутировать массивы/объекты в state
- Не понимать, что key нужен для reconciliation
- Создавать ключи на основе Math.random()
